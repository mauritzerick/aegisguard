## ğŸ”„ **Real-time Validation Flow**

### **Step-by-Step:**

1. **User focuses on field**
   - Border changes to blue (focus state)

2. **User types invalid input**
   - No error yet (only validates on blur)

3. **User tabs out (blur)**
   - Field validates immediately
   - Red border appears if invalid
   - Error message shows below field
   - Submit button disabled

4. **User focuses back on field**
   - Border stays red
   - Error message stays visible

5. **User corrects input**
   - Validates on each keystroke (since error exists)
   - When valid: red border â†’ gray border
   - Error message disappears
   - Submit button enabled

6. **User submits form**
   - All fields validate one final time
   - If any invalid, focus moves to first error
   - If all valid, form submits
